## 这是什么

判断力格斗场是一个训练系统。不是课程，不是教程。是一个训练场。

它的核心命题很简单：**知识素材是弹药库，判断力是在战场上练出来的。不上战场的人，有再多弹药也不会打仗。**

AI平权了信息处理能力之后，个体的不可替代性不在于知道多少，而在于能否在信息不完备、时间有限、后果真实的情况下做出一个经得起审视的决策。

## 快速上手（3步开始第一次训练）

### 第一步：克隆
```bash
git clone https://github.com/sqzy1314520/judgment-arena.git
cd judgment-arena
```

### 第二步：初始化训练记录
```bash
python scripts/judgment.py init
```

### 第三步：选一个案例开始练

打开 `html/index.html`（浏览器直接跑，无需后端），或者如果你是 Hermes Agent 用户，加载 SKILL 后输入"训练判断力"。

从 **15个普遍性案例** 中选一个，按流程走一遍：
1. 读场景 → 2. 做判断 → 3. 看三刀对撞 → 4. 回答苏格拉底追问 → 5. 认知复盘 → 6. 保存记录

**首次建议选"该不该跳槽"或"该不该换城市"——这两个场景最贴近日常。**

---

### 训练模式

| 模式 | 适用 | 流程 | 预计 |
|:-----|:-----|:-----|:----:|
| **快速模式** | 首次体验、时间紧 | 案例→判断→三刀精简→1个追问→复盘 | ~5分钟 |
| **标准模式** | 深度训练 | 完整7步+反方保护协议+2-3个追问+记录 | ~15分钟 |

训练中可用控制命令：`/skip` 跳过当前追问 · `/back` 回到上一步 · `/status` 查看进度

---

### 三种训练方式

| 方式 | 入口 | 说明 |
|:-----|:-----|:------|
| 🥊 **案例格斗** | 选"来一个案例" | 读取内置案例→做判断→三刀解剖→追问→复盘 |
| 📖 **勇气账复盘** | 选"勇气账复盘" | 提交你自己的真实决策→教练回溯→三刀反推当时盲区 |
| 📊 **训练记录** | 选"训练记录" | 查看进度、盲区趋势、类型覆盖分布 |

**勇气账复盘结束后，系统会问你"要不要把这个决策存为训练案例"——说"好"，它就自动转化为标准案例，下次训练可选。案例库会在你的使用中自己变厚。**

### 案例库（15个，4种生活原型）

| 原型 | 数量 | 例子 |
|:-----|:----:|:-----|
| 安全vs成长 | 5个 | 跳槽 · 换城市 · 离婚 · 转行 · 要不要二孩 |
| 人情vs利害 | 4个 | 借钱 · 发现同事错说不说 · 被甩锅 · 拉黑 |
| 需要vs想要 | 3个 | 双十一 · 换车 · 囤书 |
| 花钱买安心 | 3个 | 报班 · 考公 · 考证 |

每次训练，系统会根据你的训练历史推荐你练得最少的类型。

---

### 案例-模型绑定

不同决策类型，训练时三把刀的权重自动调整：

| 案例类型 | 加重哪把刀 | 追问方向 |
|:---------|:----------|:---------|
| 安全vs成长 | 时间折叠人 | 拉到5年10年后回头看 |
| 人情vs利害 | 换位者 | 增加多个固定角色位 |
| 需要vs想要 | 反事实者 | 剥掉欲望挖底层需求 |
| 花钱买安心 | 反事实者 | 追问"财富买不到的是什么" |

## 三把刀

| 刀 | 核心追问 |
|:---|:---------|
| **时间折叠** | 3天/3月/3年后，这个判断还站得住吗？ |
| **换位** | 换成别人（老板/执行者/反对者），他会怎么选？ |
| **反事实** | 你依赖的核心假设如果不成立，判断还成立吗？ |

## 苏格拉底追问（五类）

| 类型 | 瞄准什么 |
|:-----|:---------|
| **澄清** | 你说的概念到底是什么 |
| **假设检验** | 你依赖了什么没说的前提 |
| **证据检验** | 你的断言有依据吗 |
| **视角转换** | 你只看到了一个角度 |
| **后果追踪** | 你推到第几步发现自己还没想好 |

## 三大知识体系支撑

本训练体系的设计吸收了三个方向的研究成果：

| 体系 | 回答的问题 | 在我们设计里的落地 |
|:-----|:-----------|:-------------------|
| **CFAR（应用理性训练）** | 怎么练 | 教练定位、圆桌对撞、回映机制 |
| **学术JDM（判断与决策心理学）** | 为什么有效 | 双系统（裸判→系统1，追问→系统2）、去偏差训练 |
| **自然主义决策（Gary Klein）** | 训练什么 | 影子对手（三把刀）、事前检讨法、实战案例 |

## 21天三阶段

| 阶段 | 次数 | 你在经历什么 |
|:-----|:----:|:-------------|
| **看见** | 1-7次 | 裸判。你根本不知道自己的盲区在哪 |
| **理解** | 8-14次 | 开始能预判"时间折叠人会怎么说" |
| **内化** | 15-21次 | 还没等三刀开口，你自己先说了一遍 |

## 理论根基

这个工具不是设计出来的。是一次对话、一次纠正、一次调研、一次推翻——反复碰撞出来的。

从一个人问了一个问题开始。然后回答。然后方向对了。然后走错了，被拉回来。然后查了资料，发现了三个真正在做这件事的知识体系。然后推翻了最开始的设计。重来。再被纠正。再来。

每一处设计——为什么是三把刀、为什么是这五类追问、为什么是21天、为什么是圆桌而不是展板——都是在碰撞中才想清楚的。

完整的诞生记录见 `docs/birth-of-a-tool.md`。

### 案例是怎么来的

内置的15个案例，全部由 **案例自生长引擎**（cognitive-case-template.md）驱动生产。引擎定义了4种生活原型与认知框架的映射关系——安全vs成长→认知主权、人情vs利害→决策框架、需要vs想要→第一性原理、花钱买安心→锋刃标准——框架的每个维度自动投射到三刀和追问上，场景换一层皮，框架不变。

**外部用户提交勇气账复盘时，引擎会自动为你生成结构化案例**——你只需描述你的困境，系统自动识别原型、匹配框架、生成三刀追问。不需要阅读认知主权/决策框架/第一性原理的任何原文。引擎模板（references/cognitive-case-template.md）已提取了框架的DNA。十几页的思维体系文档不打包进SKILL，但引擎让它们的思维内核在每一次案例生产中自动运行。新的结构化案例存于 `case-library/user-cases/`，后续训练自动加载。
