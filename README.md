## 这是什么

判断力格斗场是一个训练系统。不是课程，不是教程。是一个训练场。

它的核心命题很简单：**知识素材是弹药库，判断力是在战场上练出来的。不上战场的人，有再多弹药也不会打仗。**

AI平权了信息处理能力之后，个体的不可替代性不在于知道多少，而在于能否在信息不完备、时间有限、后果真实的情况下做出一个经得起审视的决策。

## 怎么练

每一轮训练的流程：

```
1. 给你一个真实案例（信息不完备，选项各有利弊）
2. 你先做判断（红线：不能先看分析）
3. 三把刀上场辩论你的案例（圆桌对撞）
4. 反方保护协议（"我们漏了什么？"）
5. 苏格拉底追问（用问题让你自己发现盲区）
6. 认知复盘（你发现了几个自己没想到的角度？）
7. 记录趋势（10次后出报告）
```

21次为一个周期，分三个阶段：

| 阶段 | 次数 | 你在经历什么 |
|:-----|:----:|:-------------|
| **看见** | 1-7次 | 裸判。你根本不知道自己的盲区在哪。每一次对撞都让你发现"原来还可以这么看" |
| **理解** | 8-14次 | 你开始能预判"时间折叠人会怎么说"。框架开始进入你的工具箱 |
| **内化** | 15-21次 | 还没等三把刀开口，你自己先说了一遍。框架已经内化成默认思考 |

## 三把刀

| 刀 | 核心追问 |
|:---|:---------|
| **时间折叠** | 3天/3月/3年后，这个判断还站得住吗？ |
| **换位** | 换成别人（老板/执行者/反对者），他会怎么选？ |
| **反事实** | 你依赖的核心假设如果不成立，判断还成立吗？ |

## 苏格拉底追问（五类）

| 类型 | 瞄准什么 |
|:-----|:---------|
| **澄清** | 你说的概念到底是什么 |
| **假设检验** | 你依赖了什么没说的前提 |
| **证据检验** | 你的断言有依据吗 |
| **视角转换** | 你只看到了一个角度 |
| **后果追踪** | 你推到第几步发现自己还没想好 |

## 三大知识体系支撑

本训练体系的设计吸收了三个方向的研究成果：

| 体系 | 回答的问题 | 在我们设计里的落地 |
|:-----|:-----------|:-------------------|
| **CFAR（应用理性训练）** | 怎么练 | 教练定位、圆桌对撞、回映机制 |
| **学术JDM（判断与决策心理学）** | 为什么有效 | 双系统（裸判→系统1，追问→系统2）、去偏差训练 |
| **自然主义决策（Gary Klein）** | 训练什么 | 影子对手（三把刀）、事前检讨法、实战案例 |

## 使用方式

### 方式一：Hermes Agent SKILL

加载此SKILL后，输入"训练判断力"即可进入。

### 方式二：独立HTML入口

打开 `html/index.html`，浏览器运行，无需后端。

### 方式三：命令行状态管理

```bash
python scripts/judgment.py init      # 开始训练
python scripts/judgment.py status    # 当前进度
python scripts/judgment.py log       # 记录一次训练
python scripts/judgment.py stats     # 统计数据
python scripts/judgment.py trends    # 盲区趋势
```

## 项目背景

这个工具不是设计出来的。是一次对话、一次纠正、一次调研、一次推翻——反复碰撞出来的。

从一个人问了一个问题开始。然后回答。然后方向对了。然后走错了，被拉回来。然后查了资料，发现了三个真正在做这件事的知识体系。然后推翻了最开始的设计。重来。再被纠正。再来。

每一处设计——为什么是三把刀、为什么是这五类追问、为什么是21天、为什么是圆桌而不是展板——都是在碰撞中才想清楚的。

完整的诞生记录见 `docs/birth-of-a-tool.md`。
